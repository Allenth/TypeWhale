# TypeWhale 设计规范

本文档是 TypeWhale UI 设计和实现的约束来源。任何主窗口布局改动，在进入代码评审或打包测试前，都必须符合这里的尺寸、层级、组件和验收规则。

## 产品定位

TypeWhale 是 macOS 14+ 上的快速语音输入工具。主窗口是一个紧凑的桌面工作台，不是仪表盘、落地页，也不是移动端设置页。

界面需要具备以下感受：

- 快：按下快捷键后，录音状态立即出现，并且状态切换稳定。
- 安静：设置和诊断信息是辅助内容，不能压过当前语音输入流程。
- 原生：遵守 macOS 的密度、控件、焦点、弹窗、菜单和快捷键习惯。
- 可信：权限、模型、识别、失败原因、费用统计都必须可读、明确、可恢复。

## 布局模型

主窗口采用固定尺寸的 macOS workbench 三栏结构：

- 左侧栏：品牌、当前模型、权限、使用方法、版本历史、测试日志入口。
- 中间工作区：当前会话、实时文本、最近转录历史。
- 右侧 inspector：快捷键和稳定偏好设置。

中间工作区是主界面核心。任何布局调整都不能让中间区看起来比两侧区域更次要。

## 主窗口尺寸预算

目标固定内容尺寸：

- 窗口内容：`1080 x 640`。
- 左侧栏：`200`。
- 列间距：`16`。
- 右侧 inspector：`300`。
- 右外边距：`18`。
- 中间工作区：约 `530`。
- 顶部间距：中间和右侧为 `18`，左侧品牌区为 `28`。
- 底部间距：中间和右侧为 `20`，左侧为 `16`。

任何区域都不能依赖不可见溢出。如果内容无法放入这个预算，应减少、重组或把次级功能移入弹窗，而不是继续压缩主流程。

## 区域规则

### 左侧栏

- 内部水平边距：`16`。
- 品牌图标：`48 x 48`。
- 品牌标题：16 semibold。
- 版本文字：10 medium，次级颜色。
- 当前模型卡片必须是紧凑块；长模型名使用中间或尾部截断。
- 权限是紧凑状态列表，不是大型设置面板。
- 底部入口使用纯图标按钮，必须有 tooltip 和 accessibility label。
- 在 `640` 高度下，底部入口不能遮挡权限卡片。

### 中间工作区

- 当前会话卡片是视觉锚点。
- 当前会话卡片目标高度：`216-224`。
- 最近转录占据中间区剩余高度，并且内部滚动。
- 实时文本属于当前会话，不应成为另一个同权重卡片。
- 状态胶囊水平居中，并受卡片内部宽度约束。
- 状态胶囊固定高度：`32`。
- 状态胶囊宽度范围：`240-360`，不能超过卡片内部宽度。
- 长实时文本最多显示四行，超出时有意截断。
- 最近转录行使用固定最小高度和内部文本截断，不能撑大整个窗口。

### 右侧 Inspector

- 右侧 inspector 是辅助区，不能在视觉上重于中间工作区。
- 目标宽度：`300`。
- 分组必须清晰：快捷键、智能整理、翻译、截图、系统。
- 超出单行宽度的控件，必须改成上下结构。
- 横向控件组必须放入卡片内部宽度，并至少保留 `8` 像素余量。
- 快捷键行使用紧凑按钮；长快捷键文本必须有意截断。
- 高级设置进入 popover 或 dialog，不直接堆在主 inspector。

## 间距规范

- 窗口列间距：`16`。
- 板块间距：`12`。
- 标题到卡片间距：`6-8`。
- 卡片内边距：`10-14`。
- 列表卡片垂直内边距：`3-6`。
- inspector 行高：`30`。
- 权限行高：`34`。
- 紧凑栈间距：`4-7`。
- 标准栈间距：`8-12`。
- 不允许随手添加一次性间距，除非是明确的视觉微调。

## 圆角和表面

- 普通卡片最大圆角：`8`。
- 胶囊和 pill 可使用完全圆角，或根据高度使用 `10-16`。
- 不允许卡片套卡片。
- 不允许用一堆同权重卡片包围主流程。
- 边框使用 hairline 级别，不能形成厚重外框。
- 背景可以柔和分区，但不能把每个元素都做成盒子。

## 字体层级

- App 标题：16 semibold。
- 板块标题：12 medium，弱化颜色。
- 主状态：15 semibold。
- 正文和列表：11-12 regular 或 medium。
- 元信息和费用：9-10 medium，弱化颜色。
- 按钮文字：11-12 medium。
- 不使用随视口变化的字体大小。
- 中文标签不能被垂直裁切；固定高度行必须预留足够内边距。

## 颜色角色

- 背景：深色 macOS material，叠加层必须克制。
- 表面：低透明白色。
- 边框和分割线：低透明白色。
- 品牌色：TypeWhale 黄色。
- 成功：system green。
- 警告：system yellow 或 orange。
- 错误：system red。
- 弱化文字：secondary label color 或低透明白色。

颜色不能作为唯一状态信号。状态点旁边必须有文字。

## 组件规范

### 按钮

- 优先使用 macOS 原生 `NSButton` 样式。
- 纯图标按钮必须有 tooltip 和 accessibility label。
- 主窗口图标按钮点击区域稳定在 `24-34`。
- 不允许把 `...` 作为可见按钮文字。需要更多操作时，使用图标按钮、菜单或 disclosure。

### 开关

- 开关统一靠行尾对齐。
- 设置含义由左侧标签说明；除非有危险或失败状态，否则主窗口不放额外说明文字。

### Popover 和 Dialog

- 使用方法、版本历史、测试日志、模型详情、提示词编辑、高级规则进入 popover 或 dialog。
- Popover 使用固定可读宽度，长内容内部滚动。
- Dialog 必须有一个主动作，并遵守平台常见取消、重置、保存顺序。

### 列表

- 列表自己负责滚动。
- 最近转录行必须保留复制操作入口。
- 长中文和英文文本在文本列内换行，并按设计行数截断。

## 状态要求

主窗口必须设计并验证以下状态：

- 空状态：没有最近转录。
- 就绪：模型已安装、权限可用、等待录音。
- 录音中：波形活跃，实时文本可能为空。
- 开头静音：不能把无效识别文本表现成成功结果。
- 整理中：进度可见，录音状态不能闪烁、塌陷或重排。
- 完成：结果可见，粘贴或复制反馈明确。
- 失败：ASR、模型、权限失败需要说明原因和恢复路径。
- 局部失败：保留成功文本，并标记失败步骤。
- 长文本：实时文本和历史列表不能重叠。
- 长中文标签：固定高度行不能裁切文字。

## 验收标准

UI 改动必须同时满足以下条件才算完成：

- `NSWindow` 尺寸和主内容尺寸一致。
- 每个固定宽度控件都放入父级宽度预算。
- 主标签没有垂直裁切。
- 控件不覆盖文字，不互相重叠。
- 中间工作区仍是视觉主区。
- 右侧 inspector 不产生横向压缩、隐藏溢出或约束冲突。
- 列表内部滚动，不撑开主窗口。
- Popover 不以混乱方式遮挡触发它的控件。
- 实现后必须截取安装版真实窗口截图，包括整窗、左侧、中间、右侧区域。
- 必须验证深色外观，因为 App 强制使用 dark Aqua。
- UI、交互、动效或反馈状态改动必须经过 `design-review` skill 或独立 subagent 复核；不能只依赖实现者自查或用户肉眼验收。
- 动效改动必须说明并验证节奏、过渡方式、是否闪烁、是否跳变、是否保留历史好体验。
- 如果用户指出历史版本体验更好，验收前必须对比历史实现或历史记录，确认没有把已验证的好体验误删。

## 胶囊动效规则

- 胶囊实时预览是用户说话过程中的确定性锚点，不是普通状态灯。
- 文字变化应恢复并保持 1.2 版本的丝滑打字机效果：首段预览做局部淡入，后续新增尾部逐字推进。
- 不允许整段文本闪烁、整段淡入、透明度反复跳变，或用硬切替代过渡。
- 不允许为避免 ASR 修正问题而删除动效；正确边界是恢复稳定架构和稳定显示层，而不是砍掉过渡。
- 当前非流式 ASR 方案必须优先使用 1.2 已验证的累计快照直出：自然停顿、静音和换气不能触发短窗口分段拼接。
- 在累计快照架构下，`CapsuleTextBuffer` 应保留 1.2 的按当前显示长度刷新策略，再对新增尾部做打字机推进；不要把 1.3 分段架构下的“按位混字”问题误判为该策略本身错误。
- “逐字出现”指视觉上的连续流动，不是机械地一个字一个字硬跳；新增字符可以使用局部短淡入来保持丝滑感。
- 胶囊空状态波形应使用 1.2 的 7 段柱状波形和较慢落音，避免把录音状态改成不熟悉的线形状态灯。
- 胶囊动效修改必须在安装版上做真实体验检查；无法录屏时至少要说明未完成录屏验证。

## 后续 UI 改动流程

1. 如果设计意图变化，先更新本文档。
2. 改代码前先计算宽高预算。
3. 尽量使用共享 token 和共享组件。
4. 构建并安装到本机。
5. 截取安装版真实截图。
6. 修掉裁切、遮挡、错位、层级错误后再提交。
7. 源码、版本历史、构建产物和本机安装版全部对齐后，才允许认为完成。

## 当前已知风险

当前横屏 UI 只是经过部分代码数据校准，仍需要按本文档做真实截图验收。安装版中发现的任何遮挡、裁切、错位或视觉重心错误，都是 UI 质量阻断项。
