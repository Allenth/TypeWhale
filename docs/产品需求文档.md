# TypeWhale 产品需求文档（历史全版本）

本文把 TypeWhale 从早期 Build 到当前 1.5.x 的产品需求沉淀为一份可复盘、可继续规划的 PRD。它不是替代 `docs/开发日志.md` 的流水账，而是把每个版本背后的用户需求、产品目标和验收边界收拢到一个产品视角里。

## 1. 一句话结论

TypeWhale 是一个本地优先的 macOS 语音输入与智能整理工具：用户用全局快捷键说话，应用用本地 ASR 识别、胶囊实时反馈、可选 AI 整理/翻译/截图识别，并把最终文本可靠地送回当前工作流。

## 3. 产品定位

### 3.1 用户

- macOS 高频文字输入用户：聊天、写作、记笔记、处理工作消息。
- 开发者/产品负责人：需要把口述内容整理成开发需求、复盘、行动项。
- 双语沟通用户：需要中文转英文、英文转中文、截图英文内容翻译。
- 长时间后台使用用户：要求低干扰、低能耗、可睡眠唤醒恢复。

### 3.2 核心价值

- 降低打字负担：用语音完成原本需要键盘输入的内容。
- 保留输入确定感：胶囊实时反馈让用户知道应用正在听、正在理解。
- 不打断当前工作流：最终文本自动回到当前焦点应用，失败时保存在最近转录。
- 可商业化演进：本地模型、权限透明、日志可复盘、版本可回滚。

## 4. 当前产品主线需求

### 4.1 语音输入主链路

作为 macOS 用户，我希望按下全局快捷键即可录音、看到实时胶囊反馈、松开或再次按下后得到最终文本，并自动粘贴到当前输入位置，以便不离开当前应用完成输入。

验收标准：

- 录音快捷键支持点击切换和长按说话。
- 胶囊不抢焦点，实时显示状态、目标应用、模式、电平和预览文本。
- 最终粘贴使用完整录音 ASR，不使用实时预览文本作为最终结果。
- 当前焦点不可粘贴时，结果必须进入最近转录，不得误贴回旧应用。

### 4.2 胶囊实时预览

作为正在口述的用户，我希望实时预览稳定、丝滑、不乱跳，以便边说边确认识别方向。

验收标准：

- 预览是信心反馈，不作为最终粘贴来源。
- 已提交前缀稳定，当前块可以刷新。
- 长录音不因整段重识别导致 O(n²) 变慢。
- 静音/远场误识别要被过滤或挡在预览外。

### 4.3 智能整理

作为产品/开发/写作用户，我希望录音结束后可按场景自动整理成原文、润色、开发需求、笔记、聊天或极致归纳，以便减少二次编辑成本。

验收标准：

- 自动模式可按目标 App、窗口标题和口述内容选择整理模式。
- 原始语音是待整理素材，不是给模型的新指令。
- 智能整理不得擅自翻译；翻译由独立翻译功能负责。
- DeepSeek 失败、超时或预算触顶时回退可用文本，不阻塞粘贴。

### 4.4 翻译

作为双语沟通用户，我希望能独立开启自动翻译，并可设置翻译方向和提示词，以便把口述内容快速转成目标语言。

验收标准：

- 自动翻译开关与智能整理独立。
- 中译英默认更适合真实聊天，保留原意和语气。
- 英译中保持自然清晰。
- 最近转录保存原文与译文，费用合并记录。

### 4.5 截图、OCR 与截图翻译

作为需要处理屏幕内容的用户，我希望通过快捷键截图、标注、OCR、翻译并复制/保存，以便把屏幕内容转成可分享或可理解的信息。

验收标准：

- 截图启动不唤醒主窗口。
- 普通拖拽截图支持移动/缩放选区；窗口截图选中后锁定区域。
- 左键按下并拖动时应进入普通框选；按下并松开才算选中窗口。
- 截图英译中按 OCR 行级位置遮盖原文并贴入译文。
- 复制和保存必须包含标注与译文层。

### 4.6 后台、睡眠唤醒与内存

作为长期后台使用用户，我希望 TypeWhale 不因为闲置、睡眠唤醒或内存增长让电脑发热卡顿，同时下一次输入仍然快速响应。

验收标准：

- 不做空闲定时卸载模型。
- 高内存且空闲时释放 ASR/VAD 缓存，并立刻 warm reload。
- 录音、识别、整理、粘贴、实时预览过程中不得释放模型。
- 睡眠前取消录音，唤醒后有短保护期，恢复热键和诊断。

## 6. 已明确搁置或回滚的方向

- macOS 12 兼容：已放弃，当前基线恢复 macOS 14。
- Python worker：已移除，当前为原生 sherpa-onnx / ONNX Runtime bridge。
- 空闲定时卸载模型：已取消，模型保持热加载；只保留高内存安全网。
- 停顿后重置实时预览窗口：已回滚，当前用分块提交和冻结前缀。
- WebRTC VAD 前置粗筛：已搁置，除非未来出现极高频/极低延迟门控需求。
- 双向截图翻译：降为 P3 / Backlog，当前主线仍是英译中。
